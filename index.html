Tackle is a package manger for the Tcl programming language.

$ tackle search tut tutstack v1.0.0 An example Tcl package that implements a simple data structure Package URL: https://github.com/tacklepkg/packages/raw/master/tutstack/tutstack.tar.gz $ tackle add tutstack Added package tutstack v1.0.0 from https://github.com/tacklepkg/packages/raw/master/tutstack/tutstack.tar.gz to /home/username/.local/share/tackle/tutstack $ tackle ls tutstack $ tackle rm tutstack Removed tutstack.

The goal of Tackle is to help Tclers share and collaborate.

Install

Download the latest release and put tackle in your $PATH. For modern Linux distributions a good place is ~/.local/bin/tackle. Alternatively, clone the repository and $ make install.

Tackle requires that you have Tcl version 8.6 or greater already installed. Some packages may require Make and similar standard build tools.

Configure

Tackle does not require any configuration to manage packages, but Tcl will not automatically source those packages unless the Tackle install path is added to the environment variables $TCLLIBPATH and $TCLX_Y_TM_PATH.

So assuming Tcl version 8.6, for the Fish shell run this command:

$ set -Ux TCLLIBPATH $TCLLIBPATH ~/.local/share/tackle $ set -Ux TCL8_6_TM_PATH $TCL8_6_TM_PATH ~/.local/share/tackle

For Bash edit your ~/.profile:

TCLLIBPATH=$TCLLIBPATH:~/.local/share/tackle TCL8_6_TM_PATH=$TCL8_6_TM_PATH:~/.local/share/tackle

Use

Tackle tracks what should be installed in ~/.config/tackle/tracker.tackle. This file is automatically modfied whenever packages or modules are added or removed.

As demonstrated above, Tackle has a few different commands:

Command Description
$ tackle package search $query Search for available packages by $query
$ tackle package add $names Install packages by $names
$ tackle package rm $names Uninstall packages by $names
$ tackle package ls List installed packages
$ tackle package show $name Detail an installed package by $name
