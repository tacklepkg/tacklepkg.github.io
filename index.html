Tackle is a package manger for the Tcl programming language.

$ tackle add tutstack Installed tutstack from https://api.github.com/repos/tacklepkg/tutstack/tarball/v1.0.0 to /home/username/.local/share/tackle/tacklebox/tacklepkg-tutstack-9a0f89d. $ tackle ls tutstack $ tackle rm tutstack Removed tutstack.

Install

Download the latest release binary for your operating system and architecture, then place that in your $PATH. For modern Linux distributions a good place is ~/.local/bin.

Configure

Tackle does not require any configuration to manage packages, but Tcl will not automatically source those packages unless the tacklebox is added to the environment variable $TCLLIBPATH.

For the Fish shell run this command:

$ set -Ux TCLLIBPATH $TCLLIBPATH ~/.local/share/tackle/tacklebox

For Bash edit your ~/.profile:

TCLLIBPATH=$TCLLIBPATH:~/.local/share/tackle/tacklebox

Tackle installs packages from a central package repository by default, but can install any Tcl package available as a tarball over HTTP: it is a fundamentally decentralized system.

Use

Tackle tracks what packages should be installed in ~/.config/tackle/packages.tackle. This file can be edited manually, and is automatically modfied whenever packages are added or removed.

As demonstrated above, Tackle has a few different commands:

  • The add command takes the name and optionally the URL of a package as arguments. If no URL is given it looks for the name of the package in the official repositories.

  • The rm command takes any number of package names and deletes them all from the system.

  • The ls command optionally takes a globbing pattern and lists installed packages matching that pattern. Without a pattern argument is lists all installed files.

  • The sync command takes no arguments and adds or removes packages from the tacklebox as necessary given what is specified in the packages.tackle file.

If no command is specified, Tackle defaults to sync.
