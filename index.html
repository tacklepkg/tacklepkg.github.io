Tackle is a package manger for the Tcl programming language.

$ tackle add tutstack Installed tutstack from https://raw.githubusercontent.com/tacklepkg/packages/ master/tutstack/tutstack.tar.gz to /home/username/.local/share/tackle/ packages/tutstack.tcl. $ tackle ls tutstack $ tackle rm tutstack Removed tutstack.

Tackle is designed to be simple, flexible, and fastâ€”just like Tcl. Both pkgIndex.tcl defined packages and tm modules are supported.

The goal of Tackle is to help Tclers share and collaborate.

Install

Download the latest release and put tackle in your $PATH. For modern Linux distributions a good place is ~/.local/bin/tackle. Tackle requires that you have Tcl already installed.

Configure

Tackle does not require any configuration to manage packages, but Tcl will not automatically source those packages unless the install paths are added to the environment variable $TCLLIBPATH and $TCLX_Y_TM_PATH.

For the Fish shell run this command:

$ set -Ux TCLLIBPATH $TCLLIBPATH ~/.local/share/tackle/packages $ set -Ux TCL8_6_TM_PATH $TCL8_6_TM_PATH ~/.local/share/tackle/modules

For Bash edit your ~/.profile:

TCLLIBPATH=$TCLLIBPATH:~/.local/share/tackle/packages TCL8_6_TM_PATH=$TCL8_6_TM_PATH:~/.local/share/tackle/modules

Tackle installs packages from a central repository by default, but can install any Tcl package available as a tarball over HTTP: it is a fundamentally decentralized system.

Use

Tackle tracks what should be installed in ~/.config/tackle/packages.tackle and ~/.config/tackle/modules.tackle. These files can be edited manually, and are automatically modfied whenever packages or modules are added or removed.

As demonstrated above, Tackle has a few different commands:

  • The add command takes the name and optionally the URL of a package as arguments. If no URL is given it looks for the name of the package in the official repositories.

  • The rm command takes any number of package names and deletes them all from the system.

  • The ls command optionally takes a globbing pattern and lists installed packages matching that pattern. Without a pattern argument is lists all installed files.

  • The sync command takes no arguments and adds or removes packages as necessary given what is specified in the .tackle file.

All of the package commands and module commands work the same way, except for add:

  • $ tackle package add $name $url expects a URL pointing to a tarball.
  • $ tackle module  add $name $url expects a URL pointing to a .tm file.

If no command is specified, Tackle runs sync for both packages and modules.

Contribute
